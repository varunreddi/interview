*) Component Scanning | basePackage Rule:-
> Starter class (or) main class will be generated by Spring Initializer.
  It contains one package name [starting line]

ex: in.nareshit.raghu

> All your classes (that we create next after to project) must be inside
 same package [ex: in.nareshit.raghu] or its sub-package [ex: in.nareshit.raghu.model]
 then only Spring container can detect you class and supports for object creation.

---In Spring f/w[Not Spring boot]---------------------------------------
@ComponentScan(basePackages="in.nareshit.raghu")
class AppConfig { }

//Spring container , please scan your classes from package: in.nareshit.raghu
ApplicationContext ac = new AnnotationConfigApplicationContext(AppConfig.class);

---FAQs-------------------------------------------------------------
Q) Can we modify default package naming rule for @ComponentScan given by
		Spring Boot?

A) YES. But not recomanded.
   We can write our package name at main class : 
    ex: @ComponentScan("com.app.test")

 **** If you provide externally then main class package is no longer behaves like
	basepackage.


Q) Can we provide multiple package as basePackage externally?
A) YES. But not recomanded.

At main class level:
@ComponentScan({
	"com.app.test",
	"in.nareshit.raghu"
})

Q) What is @Repeatable annotation in java ?
A)

*) Note: @ComponentScan is a Repeatable Annotation, that means
          we write this annotation multiple times at main class
	  with different package names.
